# APPLY_ORDER

Apply this repository in the following order.

## Phase 1 — Core operating layer

1. Apply `workflows/skill-vetting.md`
2. Apply `workflows/project-state-tracking.md`
3. Apply `workflows/session-wrap-up.md`

## Phase 2 — Delegation quality

4. Apply `workflows/multi-agent-handoff-review.md`
5. Review `examples/handoff-test/`

## Phase 3 — Daily operations

6. Apply `workflows/morning-brief.md` in manual mode first
7. Only after a successful manual trial, add scheduling

## Phase 4 — Skill adoption

8. Review `skills/registry.md`
9. Read `sources/awesome-openclaw-skills.md` and `sources/awesome-openclaw-usecases.md`
10. Vet candidate skills before installation

## Rule

Do not jump straight to automation or random skill installs before the core operating layer is in place.
