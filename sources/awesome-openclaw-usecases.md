# Source Notes: awesome-openclaw-usecases

## Source
- https://github.com/hesamsheikh/awesome-openclaw-usecases

## Why this source matters
- shows real operating patterns instead of only tool catalogs
- useful for deciding what to automate next

## Key takeaways
- workflow quality matters more than raw skill count
- morning briefs, project state, and delegation patterns are high-leverage
- goal-driven autonomy works best when task generation is constrained and observable
- overnight automation should produce inspectable local artifacts before any deployment
- manual proving should come before automation

## Patterns pulled from this source
- custom morning brief
- project state management
- goal-driven autonomous task loops
- overnight mini-app building lane
- n8n workflow orchestration
- semantic memory search
- self-healing / maintenance operating patterns

## How to use this source
1. choose a concrete operational pain point
2. find a matching use case pattern
3. adapt it into a local workflow
4. automate only after it proves useful manually
