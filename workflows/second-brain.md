# Workflow: Second Brain

## Purpose

Create a low-friction capture system for ideas, reminders, links, and notable facts.

## When to use

- when capture must be as easy as messaging the agent
- when the user wants retrieval without heavy manual organization

## Inputs

- free-form messages from chat
- optional links or references

## Outputs

- stored memory entry
- optional categorized note or later dashboard-ready record

## Procedure

1. Accept capture in plain language with minimal friction.
2. Store durable items in memory files.
3. If the item affects active work, also update project state.
4. Keep later retrieval in mind: include enough wording to be searchable.
5. Treat dashboards as optional later UI, not the first requirement.

## Verification

- capture is fast enough to use often
- stored items are later retrievable
- active-work items do not get lost in general memory only

## Risks / gotchas

- capture systems fail when they require too much structure up front
- not every thought belongs in long-term memory
