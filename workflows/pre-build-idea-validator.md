# Workflow: Pre-Build Idea Validator

## Purpose

Check competitive reality before spending time building a new project, feature, or tool.

## When to use

- before starting a new product idea
- before large new feature work
- before building something that may already be crowded

## Inputs

- idea description
- optional target audience or niche

## Outputs

- competition snapshot
- crowded / moderate / open judgment
- proceed / pivot / stop recommendation

## Procedure

1. Search GitHub for direct and adjacent competitors.
2. Search the web for Product Hunt, npm, PyPI, Hacker News, and obvious market players.
3. Record the top competitors and what they already cover.
4. Decide whether the space is:
   - crowded
   - moderate
   - relatively open
5. If crowded, propose 2-3 differentiation angles before building.
6. Do not start implementation until the reality check is shown.

## Verification

- top competitors are named
- market crowding is stated explicitly
- proceed/pivot/stop recommendation is clear

## Risks / gotchas

- this is a gate, not a perfect market oracle
- a crowded space can still be viable if the angle is strong
- a low-competition space can still mean weak demand
