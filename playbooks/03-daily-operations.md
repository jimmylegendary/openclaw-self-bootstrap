# Playbook 03 — Daily Operations

1. Keep project state current.
2. Write important daily context to memory.
3. Use session wrap-up at the end of meaningful work.
4. Trial morning briefs manually before scheduling them.
5. Use a visible loop: planning -> execution -> optional overnight build -> next-morning surfacing.
6. Keep paths predictable:
   - `ops/autonomous-goals.md`
   - `memory/tasks-log.md`
   - `projects/overnight/YYYY-MM-DD-slug/`
